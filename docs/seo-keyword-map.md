## Keyword Clusters

### 1. URL / Web to EPUB (core)

| Item | Detail |
|---|---|
| **Target keywords** | "url to epub", "web to epub", "webpage to epub", "convert url to epub" |
| **Search intent** | Transactional — user has a URL and wants an EPUB file now |
| **Current page** | `/` (homepage — converter tool) |
| **Recommended expansion** | None needed for core; homepage already serves this directly |
| **Priority** | **P1** — primary traffic driver |

### 2. Article / Blog Post to EPUB

| Item | Detail |
|---|---|
| **Target keywords** | "article to epub", "save article to epub", "blog post to epub", "read later epub" |
| **Search intent** | Transactional with "read later" flavour — wants to save specific content types |
| **Current page** | `/` (served by how-it-works copy and FAQ) |
| **Recommended expansion** | A short `/use-cases` or `/how-it-works` page covering specific content types (articles, docs, recipes) with examples |
| **Priority** | **P1** — high-volume variant of core use case |

---

### 3. Kindle-specific conversion

| Item | Detail |
|---|---|
| **Target keywords** | "convert webpage to kindle", "send article to kindle epub", "kindle epub from url" |
| **Search intent** | Transactional — Kindle owner looking for an easy workflow |
| **Current page** | `/` (FAQ answer covers Kindle explicitly) |
| **Recommended expansion** | A `/send-to-kindle` guide page: step-by-step walkthrough of download → Send-to-Kindle flow |
| **Priority** | **P1** — Kindle is by far the most common e-reader brand; dedicated content captures brand-specific queries |

---

### 4. Kobo / generic e-reader conversion

| Item | Detail |
|---|---|
| **Target keywords** | "kobo epub converter", "sideload epub kobo", "web page to kobo" |
| **Search intent** | Transactional — Kobo owner, same job-to-be-done as Kindle cluster |

---

### 5. Free / no-account angle

| Item | Detail |
|---|---|
| **Target keywords** | "free epub converter", "epub converter no signup", "epub converter no account" |
| **Search intent** | Informational/transactional — user is comparison-shopping and wants friction-free option |
| **Current page** | `/` (FAQ explicitly answers this; body copy mentions "No account required"; OG/Twitter description mentions "Free and instant") |
| **Recommended expansion** | "Free" does not appear in the `<title>` tag. Consider adding it: e.g. "EpubAnything — Free URL to EPUB Converter". Also confirm OG description includes both "free" and "no account" in the same sentence for snippet clarity. |
| **Priority** | **P2** — good differentiator from competitors that require accounts |

---

### 6. Documentation / technical reading

| Item | Detail |
|---|---|
| **Target keywords** | "save documentation to epub", "read docs offline epub", "docs to epub" |
| **Search intent** | Informational — developer or technical reader wanting offline access to docs |
| **Current page** | `/` (mentioned briefly in body copy) |
| **Recommended expansion** | A `/use-cases/documentation` or blog post: "How to save any docs site to EPUB for offline reading" — targets long-tail with real narrative |
| **Priority** | **P3** — smaller volume but high-quality, technically-engaged audience |

---

## Notes

- All P1 work is already partially served by the homepage. The FAQ entries directly address the three highest-intent clusters (core conversion, Kindle, free/no-account).
- First expansion to build: a `/send-to-kindle` guide page. It targets the highest-volume brand-specific query and creates an internal link from the homepage FAQ answer.
- Avoid thin keyword-stuffing pages. Each future page should answer a real question or walk through a real workflow.
